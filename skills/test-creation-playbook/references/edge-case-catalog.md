# 邊緣案例目錄

建立測試情境時，請對照下方五大類別逐項篩選。不是每個類別都會適用當前功能，但**每個功能都應該跑過一次這份清單**。

---

## 1. 資料型別與邊界值

### 數值
- `0`、負數、`-1`（常見 off-by-one 邊界）
- 小數點：`0.5`、`0.01`、`0.001`（小數精度）
- 超出整數上限的 ID（`Number.MAX_SAFE_INTEGER`、`PHP_INT_MAX`）
- `NaN`、`Infinity`、`-Infinity`

### 字串
- 空字串 `""`、全空白 `"   "`、只有 tab / 換行
- 超長字串（1MB 以上）
- Unicode、Emoji、RTL 文字（阿拉伯語、希伯來語）
- 特殊字元：`"`、`'`、`<`、`>`、`&`、`\`、`\n`、`\r\n`
- 控制字元（`\0`、`\x1B`）

### Email
- `user@@domain..com`（雙 @ / 雙 dot）
- `user+tag@domain.com`（plus sign）
- 超長 local part（64 字以上）
- 全空白 email
- 沒有 TLD 的 email

### 日期/時間
- UTC 與本地時區切換點
- 剛好在到期時間點刷新
- 閏年 `2/29`、非閏年 `2/29`（應該失敗）
- 夏令時間切換（spring forward / fall back）

### 其他型別
- `null`、`undefined`、空陣列 `[]`、空物件 `{}`
- 型別錯誤（字串傳給數字參數）
- 布林邊界：`0`、`""`、`null` 的 truthy/falsy 判斷

---

## 2. 業務邏輯邊界

### 庫存/數量
- 負數庫存下單
- 小數點庫存（`0.5` 件）
- `0` 件加入購物車
- 超過庫存的數量
- 多人同時搶最後一件

### 金額/價格
- `null` 價格、`0` 價格、負數價格
- 四捨五入邊界（`0.005`、`0.015`）
- 幣別轉換後的精度遺失

### 狀態轉換
- 已刪除資源的存取殘留
- 已取消訂單的再次操作
- 狀態機非法轉換（從「已出貨」退回「處理中」）

---

## 3. 並發與時序

- 同一操作在兩個 tab 同時執行
- 多使用者同時編輯同一資源（樂觀鎖 / 悲觀鎖）
- 搶購：多人同時下單最後一件
- Request 順序顛倒（後發先至）
- 快取失效與實際資料不一致
- 事務未提交時的中斷

---

## 4. 安全性

### Injection
- XSS 輸入：`<script>alert(1)</script>`、`"><img src=x onerror=...>`
- SQL Injection：`' OR 1=1--`、`'; DROP TABLE users;--`
- Path Traversal：`../../../etc/passwd`
- Command Injection：`; rm -rf /`、`| cat /etc/passwd`

### 權限邊界
- 使用他人 ID 存取其資源（IDOR）
- 低權限使用者呼叫高權限 endpoint
- 登出後 token 殘留可用
- CSRF（非同源 request 帶 credential）

### 輸入驗證
- Content-Type 偽造（上傳 `.exe` 改副檔名為 `.jpg`）
- 上傳超大檔案耗盡磁碟
- 巢狀 JSON 爆炸（`{a:{a:{a:...}}}` 深度 10000）
- XML External Entity（XXE）

---

## 5. 國際化與環境

- 不同語系下的排序（英文、中文、日文）
- 貨幣格式（`$1,000.00` vs `1.000,00€`）
- 數字格式（`1,234` 在歐洲是 `1.234`）
- 時區差異：伺服器 UTC、使用者 `Asia/Taipei`
- 瀏覽器差異：Chrome / Firefox / Safari edge cases
- 行動裝置 viewport、觸控事件
- 網路不穩：慢網、斷網、request timeout

---

## 組合矩陣思維

單元 / 整合測試不要只測單一維度，應以「**功能 × 角色 × 狀態 × 邊界值**」矩陣展開。

範例：商品下單功能

| 角色 | 商品狀態 | 數量邊界 | 預期 |
|---|---|---|---|
| 一般會員 | 已上架 | `1`（正常） | 成功 |
| 一般會員 | 已上架 | `0` | 失敗（驗證錯誤） |
| 一般會員 | 已上架 | `-1` | 失敗（驗證錯誤） |
| 一般會員 | 已下架 | `1` | 失敗（商品不可購買） |
| 一般會員 | 庫存 0 | `1` | 失敗（庫存不足） |
| 訪客 | 已上架 | `1` | 失敗（未登入） |

E2E 測試**不展開**這種矩陣，只覆蓋「一般會員 × 已上架 × 正常數量 × 成功」這條 happy path。
