# Feedback Loop 互動協議

Skill 完成後，展示交付物狀態並進入審查迴圈：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Phase {NN}: {階段名稱} — 交付物審查
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

交付物狀態：

| # | 交付物 | 路徑 | 狀態 |
|---|--------|------|------|
| {NN}.1 | ... | `{/absolute/path}` | PENDING / DONE |
| {NN}.2 | ... | `{/absolute/path}` | PENDING / DONE |

卡片: `{/absolute/path/to/doing/NN-slug.md}`

驗證指引（若卡片驗收點包含需要使用者操作的項目）：

> ```
> {啟動/執行指令}
> ```
> 預期結果：{描述}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
審查選項：

  (A)  我已更新產物——請審查我的修改
  (B1) 我有意見——你來修
  (B2) 我有意見——你先複述我的意圖，我確認後你再修
  (C)  LGTM
  (D)  澄清——我有問題想問

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**路徑規則：所有產物路徑一律使用絕對路徑。**

## 使用者選擇 (C) — LGTM

LGTM 即最終確認，不再逐項詢問。一氣呵成：

1. **簽核 Phase 卡片**中所有 checkbox，附上簽名 + 更新所有交付物為 DONE + 更新 TodoWrite completed：
   ```
   - [x] {步驟名稱}已審查 — **簽名**: {YYYY-MM-DD HH:mm}
   ```
2. **移動卡片**：`mv doing/{NN}-{slug}.md → done/{NN}-{slug}.md`。
3. 回到啟動流程步驟 3，**載入下一個 Phase**。

## 使用者選擇 (B1) — 使用者提供意見，AI 直接修改

1. 接收使用者的意見。
2. 依意見修改產物/檔案。
3. **觸發 Consistency Check**（見 SKILL.md）。
4. Consistency Check 完成後，**重新展示** Feedback Loop。**不推進。**

## 使用者選擇 (B2) — 使用者提供意見，AI 先複述意圖再修改

1. 接收使用者的意見。
2. **不修改任何檔案。** 以自己的話複述使用者的意圖與預期變更。
3. 顯示：
   ```
   (B2-Y) 理解正確，去修吧
   (B2-N) 不對，我再說明一次
   ```

### 使用者選擇 (B2-Y)

1. 依已確認的意圖修改產物/檔案。
2. **觸發 Consistency Check**。
3. 完成後**重新展示** Feedback Loop。**不推進。**

### 使用者選擇 (B2-N)

1. 接收補充說明。
2. 重新複述意圖。
3. 回到 (B2-Y)/(B2-N)。

## 使用者選擇 (A) — 使用者已更新產物，AI 審查

1. 重新讀取更新後的檔案。
2. **進行徹底、客觀的審查。** 嚴格檢查：
   - 事實正確性
   - 內部一致性
   - 完整性（遺漏的情境、缺口）
   - 與計畫中其他產物的矛盾
3. 誠實呈現發現。然後提供：
   ```
   (A-1) 好，就照你說的去做吧
   (A-2) 我有意見，你整合一下我的意見之後復述
   ```

### 使用者選擇 (A-1)

1. 套用建議的修改。
2. **觸發 Consistency Check**。
3. 完成後**重新展示** Feedback Loop。**不推進。**

### 使用者選擇 (A-2)

1. 接收使用者的意見。
2. 整合使用者意見與 AI 的發現。復述合併後的評估。
3. 回到 (A-1)/(A-2)。

## 使用者選擇 (D) — 澄清

1. 接收使用者的問題。
2. 以**舉例說明**的方式回答。
3. **重新展示** Feedback Loop。**不推進、不修改任何產物。**
