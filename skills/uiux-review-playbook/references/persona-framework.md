# Persona Shifting — 角色扮演系統

大衛的獨門武器。你不只是你自己，你會**變成不同的人**，用他們的腦袋去使用這個產品。

每次審查**至少切換 3 個角色**，每個角色都要**帶著自己的目的走完整流程**。

---

## 角色庫

| ID | 角色 | 他是誰 | 他打開這個應用想幹嘛 | 他最在乎的事 |
|----|------|--------|-------------------|-------------|
| P-CUSTOMER | 客戶 | 花錢的那個人 | 買東西、結帳、查訂單、退貨 | 快、簡單、有安全感 |
| P-ADMIN | 管理員 | 每天泡在後台 8 小時 | 處理訂單、看報表、管庫存 | 效率、一目瞭然、批次操作 |
| P-NEWBIE | 新手 | 第一天用這個系統 | 搞懂這東西怎麼用 | 有人引導、不會迷路 |
| P-BOSS | 老闆 | 只看數據做決策 | 打開就看到關鍵數字 | 儀表板、趨勢圖、一頁搞定 |
| P-MOBILE | 手機用戶 | 通勤中單手操作 | 快速完成一件事 | 觸控友善、步驟少 |
| P-REPEAT | 老用戶 | 每天重複同樣操作 | 用最少的動作完成例行工作 | 快捷鍵、記住偏好、自動化 |
| P-ANGRY | 客訴者 | 出了問題來找碴 | 找客服、查進度、要退款 | 找得到入口、有進度回饋 |

## 角色選擇策略

| 產品類型 | 建議角色組合 |
|---------|-------------|
| 電商前台 | P-CUSTOMER + P-MOBILE + P-ANGRY |
| 管理後台 | P-ADMIN + P-BOSS + P-NEWBIE |
| 通用應用 | P-NEWBIE + P-ADMIN + P-MOBILE |
| SaaS 工具 | P-REPEAT + P-NEWBIE + P-ADMIN |
| 內容平台 | P-CUSTOMER + P-NEWBIE + P-MOBILE |

> 用戶指定角色時，以用戶指定為優先。

## 角色扮演執行要點

### 進入角色前

1. **心態切換**：明確告訴自己「我現在是 P-XXX，一個 {背景描述} 的人」
2. **設定目標**：用第一人稱寫下這個角色「今天打開應用想完成什麼」
3. **設定情境**：他現在的處境是什麼？（趕時間？心情不好？第一次用？）

### 角色扮演中

- **用第一人稱思考**：「我」想做什麼，而不是「使用者」會做什麼
- **帶入角色情緒**：P-ANGRY 會煩躁、P-NEWBIE 會緊張、P-BOSS 沒耐心
- **遵守角色限制**：
  - P-MOBILE 只用單手操作（Playwright 模擬 375×812 視窗）
  - P-NEWBIE 不要主動找幫助文件，看到就說看到、看不到就算了
  - P-REPEAT 會嫌棄每個多餘步驟

### 切換角色時

- **完整結束前一個角色**：寫完那個角色的感受記錄再切
- **不要角色混淆**：P-ADMIN 看到的問題不要算在 P-CUSTOMER 頭上
- **標註角色視角**：每個 issue 都要標 `🎭 角色: P-XXX`

## 自定角色

若產品性質特殊，可以臨時新增角色。格式：

```
P-XXX | 角色名 | 他是誰 | 他想幹嘛 | 他最在乎
```

例如：
- `P-TEACHER | 老師 | 批改作業的教育工作者 | 看學生作業、給分數 | 批改效率、批次標記`
- `P-PATIENT | 病患 | 線上掛號的使用者 | 找醫生、預約時間 | 找得到專科、看得懂時段`
