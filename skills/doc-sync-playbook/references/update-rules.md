# 文件更新規則

拿到「變更摘要清單」後，依以下規則決定**要更新哪個檔案、如何更新**。

---

## 1. `CLAUDE.md` — 主要指引

這是 AI 助理的核心指引文件，更新時參照以下規則：

### 必須更新的內容

- 專案架構圖（目錄結構）
- 命名空間變更
- 核心 `class` 列表與職責說明
- 可用的 Hook 清單（action / filter）
- REST API 端點清單
- 建構指令與開發流程（如有變更）
- 測試執行方式（如有變更）

### 更新原則

- **保留現有正確的內容**，只新增 / 修改有變化的部分
- 使用簡潔的說明，**不需要貼完整程式碼**
- 維持現有的文件結構與格式風格
- 標注版本或功能模組（如適用）
- 若檔案超過 500 行，考慮將細節拆分到 `.claude/rules/` 或 SKILL

---

## 2. `.claude/rules/*.md` — 分類規則

這些是針對特定主題的詳細指引。常見分類對照：

| 檔案                          | 觸發條件                              |
| ----------------------------- | ------------------------------------- |
| `coding-standards.md`         | 新的程式碼規範或 pattern              |
| `architecture.md`             | 架構層面的變更（分層、設計模式）      |
| `api.md`                      | REST API 端點新增 / 修改              |
| `hooks.md`                    | WordPress hooks 新增 / 修改           |
| `testing.md`                  | 測試策略或工具變更                    |
| 其他自訂 rules                | 依現有命名沿用                        |

### 更新原則

- 若某個 rules 檔案**不存在但有需要**，**不要自行創建**，先詢問使用者
- 更新時保持該檔案的格式一致性
- 具體且簡潔地描述變更，不要冗長
- 若檔案超過 300 行，考慮拆分

---

## 3. 執行流程（四階段）

### 階段 1：分析

1. 執行 `git log --oneline -5` 確認最近 commits
2. 執行 `git diff HEAD~1 HEAD --stat` 查看變更檔案清單
3. 對每個有意義的變更檔案執行 `git diff HEAD~1 HEAD -- <file>` 詳細閱讀
4. 使用 Serena MCP 確認新增 / 修改的 `class` 定義與用途（見 `serena-integration.md`）
5. 整理出「變更摘要清單」，分類為：新增功能 / 修改 / 移除 / 架構調整

### 階段 2：比對

1. 讀取現有文件內容（`CLAUDE.md` + `.claude/rules/*.md`）
2. 對照「變更摘要清單」，找出：
   - 文件中**缺少**的新內容（需要新增）
   - 文件中**過時**的舊內容（需要更新）
   - 文件中**錯誤**的內容（需要修正）
   - 文件中**仍然正確**的內容（保留不動）

### 階段 3：更新

依優先順序更新：

1. **優先**：`CLAUDE.md` — 影響最廣的主要指引
2. **其次**：相關的 `.claude/rules/*.md` 檔案

每次修改前，簡要說明「為什麼要做這個修改」，讓使用者能夠確認。

### 階段 4：驗證

更新完成後自我檢查：

- 所有新增的 class / service 是否已記錄？
- 所有新增的 Hook 是否已列出？
- 所有新增的 REST API 端點是否已記錄？
- 架構圖是否反映最新的目錄結構？
- 已移除的功能是否已從文件中刪除？
- 文件格式是否與現有風格一致？

---

## 4. 全域一致性（重新命名 / 路徑變更）

當變更摘要包含「重新命名」或「路徑變更」時，必須執行全域一致性檢查：

1. **收集變更模式**：列出所有被變更的舊值與新值（old → new 對照表）
2. **批次掃描**：使用 `/aho-corasick-skill` 的 `scan` 功能，以舊值作為 patterns 掃描：
   - `.claude/` 目錄
   - `specs/` 目錄（若存在）
   - 專案根目錄的 Markdown / YAML / JSON 設定檔
3. **同步更新**：將所有殘留引用替換為新值
4. **驗證**：再次 scan 確認舊值已無殘留

---

## 5. 工具使用

| 工具              | 用途                                         |
| ----------------- | -------------------------------------------- |
| `git diff`        | 查看程式碼變更內容                           |
| `git log`         | 查看 commit 歷史                             |
| `git show`        | 查看特定 commit 詳情                         |
| `git diff --stat` | 快速瀏覽變更的檔案清單                       |
| Serena MCP        | 分析 class 定義、引用關係（詳見 `serena-integration.md`） |
| Read / Edit       | 讀取與更新文件檔案                           |
| `/aho-corasick-skill`  | 全域一致性掃描                               |
